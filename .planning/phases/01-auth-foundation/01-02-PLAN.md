  <action>
    1. Create `src/lib/auth.ts`:
       - Import `Google`, `generateState`, `generateCodeVerifier` from `arctic`
       - Export `googleClient` = new Google(GOOGLE_CLIENT_ID, GOOGLE_CLIENT_SECRET, callback URL)
       - Re-export `generateState`, `generateCodeVerifier` from arctic for convenience
       - Note: Use a SINGLE Google client. The school link flow uses a different redirect_uri, so create a second client instance `googleSchoolClient` with redirect URI pointing to `/link/school-google/callback`

    2. Create `src/app/login/google/route.ts` (GET handler):
       - Generate state + codeVerifier using arctic helpers
       - Create authorization URL with scopes: openid, email, profile, https://www.googleapis.com/auth/calendar
       - Set `access_type=offline` on the URL
       - Do NOT set `prompt=consent` (per Pitfall 3 in RESEARCH.md — only force consent when needed)
       - Store state and codeVerifier in httpOnly cookies (maxAge 10 min, sameSite lax, secure in production)
       - Return Response.redirect(url)

    3. Create `src/app/login/google/callback/route.ts` (GET handler):
       - Read state + codeVerifier from cookies, compare state to query param
       - If error query param present (e.g., access_denied), redirect to `/?error=access_denied`
       - Call `googleClient.validateAuthorizationCode(code, codeVerifier)` to get tokens
       - Use `decodeIdToken` from arctic to extract sub, email, name from idToken
       - Upsert user: SELECT by googleId, INSERT if not found
       - Store encrypted tokens: INSERT into oauthTokens with role="personal", ON CONFLICT (userId, role) UPDATE access token + expiry. CRITICAL per Pitfall 2: preserve existing encryptedRefreshToken if new tokens don't include a refresh token (hasRefreshToken() returns false)
       - Call `setSessionCookie(user.id)`
       - Delete the OAuth state/verifier cookies
       - Redirect to `/`

    4. Create `src/app/link/school-google/route.ts` (GET handler):
       - Call `getSession()` — if null, return 401
       - Generate state + codeVerifier
       - Create authorization URL with scopes: openid, email, profile, https://www.googleapis.com/auth/calendar.readonly (read-only for school per RESEARCH.md scopes table)
       - Set `access_type=offline` and `prompt=select_account` (let user pick school account)
       - Store state + codeVerifier in cookies with prefix "link_" to avoid collision with login cookies
       - Redirect to URL

    5. Create `src/app/link/school-google/callback/route.ts` (GET handler):
       - Read link_oauth_state + link_code_verifier from cookies
       - Handle error query param: redirect to `/?error=school_access_denied` (Pitfall 4: school Workspace may block OAuth)
       - Get session to identify current user
       - Exchange code for tokens using googleSchoolClient
       - Extract email from idToken for display purposes
       - INSERT into oauthTokens with role="school" for current user, ON CONFLICT update. Same refresh token preservation logic as personal callback.
       - Delete link cookies
       - Redirect to `/`

    6. Create `src/app/api/auth/signout/route.ts` (POST handler):
       - Get session — if null, return 401
       - Delete all oauthTokens rows for this userId
       - Delete user row (cascade will also clean tokens, but explicit is clearer)
       - Call clearSessionCookie()
       - Return 200 JSON { success: true }
       - Per user decision: "Sign-out clears session cookie and DB tokens, redirects to setup wizard" — the redirect happens client-side after receiving the 200
