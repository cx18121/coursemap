**gcalSync.ts refactor:**
Completely rewrite `src/services/gcalSync.ts`. Remove the old `syncToGoogleCalendar` function. Replace with:

```typescript
export interface SyncProgress {
  courseName: string;
  processed: number;
  total: number;
}
export interface SyncSummary {
  inserted: number;
  updated: number;
  skipped: number;
  failed: number;
  errors: string[];
}
export async function syncCanvasEvents(
  userId: number,
  events: CanvasEvent[],
  courseColorMap: Record<string, string>,
  onProgress?: (progress: SyncProgress) => void
): Promise<SyncSummary>
```

Implementation:
1. Call `getFreshAccessToken(userId, 'personal')` — if null, throw error
2. Create Google Calendar API client with the access token
3. Group events by courseName
4. For each course:
   a. Call `ensureSubCalendar(calendar, userId, courseName, courseColorMap[courseName] || '9')`
   b. Bulk fetch existing events: `calendar.events.list({ calendarId: subCalId, privateExtendedProperty: ['canvasSourceCalendarId=canvas'], maxResults: 2500, singleEvents: true })`
   c. Build Map of existing UID -> event for diffing
   d. For each incoming event: if UID not in existing set, insert; if UID exists but fields changed (summary, start, end, description), update; otherwise skip
   e. On each event processed, call `onProgress?.({ courseName, processed: i+1, total: courseEvents.length })`
5. Accumulate counts into SyncSummary and return
6. Do NOT set colorId on individual events — color is set at sub-calendar level only

Create `src/services/schoolMirror.ts` with:

**listSchoolCalendars(userId: number):**
- Call `getFreshAccessToken(userId, 'school')` — return empty array if null (no school account linked)
- Create Google Calendar client with school access token
- Call `calendarList.list({ minAccessRole: 'reader' })`
- Filter: skip entries where `accessRole === 'freeBusyReader'` or `id` ends with `#contacts@group.v.calendar.google.com` or `#holiday@group.v.calendar.google.com` or `#weather@group.v.calendar.google.com`
- Query `schoolCalendarSelections` for this userId
- Return `SchoolCalendar[]` with `{ calendarId: string, name: string, selected: boolean }` — `selected` is true if found in DB with enabled=true, or true by default for new calendars

**mirrorSchoolCalendars(userId: number, onProgress?: callback):**
- Get selected school calendars from DB (enabled=true)
- Get personal access token via `getFreshAccessToken(userId, 'personal')`
- Get school access token via `getFreshAccessToken(userId, 'school')`
- For each selected school calendar:
  a. Ensure mirror sub-calendar on personal account: `ensureMirrorSubCalendar(personalCalendar, userId, calendarName)` — similar to ensureSubCalendar but stores in schoolCalendarSelections.gcalMirrorCalendarId
  b. Fetch events from school calendar: `schoolCalendar.events.list({ calendarId, timeMin: 30 days ago, timeMax: 90 days ahead, singleEvents: true, maxResults: 2500 })`
  c. Bulk fetch existing mirror events from personal sub-calendar using `privateExtendedProperty: ['schoolSourceCalendarId=school']`
  d. Diff and insert/update (same pattern as Canvas sync)
  e. Set `extendedProperties.private.schoolEventId` = school event ID for dedup
  f. Do NOT clean titles — copy summary as-is from school calendar
- Return `MirrorSummary` (same shape as SyncSummary)

Also add `ensureMirrorSubCalendar` helper to `gcalSubcalendars.ts` — similar to `ensureSubCalendar` but uses `schoolCalendarSelections.gcalMirrorCalendarId` and summary prefix "School - ".
