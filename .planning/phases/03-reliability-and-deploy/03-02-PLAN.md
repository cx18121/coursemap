  <name>Task 1: Deploy to Vercel via CLI</name>
  <files>vercel.json</files>
  <action>
1. Verify the project builds cleanly: `npm run build`
   - If build fails, fix errors before proceeding.

2. Ensure the GitHub repo is up to date:
   - `git add -A && git status` to check for uncommitted files
   - Push to GitHub if needed: `git push origin main`

3. Install Vercel CLI if not present: `npm i -g vercel`

4. Link project to Vercel: `vercel link`
   - Follow prompts to create a new Vercel project or link to existing

5. Create `vercel.json` at project root (if not already present) with minimal config:
   ```json
   {
     "$schema": "https://openapi.vercel.sh/vercel.json",
     "framework": "nextjs"
   }
   ```

6. Set environment variables via CLI (the user will need to provide values):
   ```bash
   vercel env add DATABASE_URL production
   vercel env add TOKEN_ENCRYPTION_KEY production
   vercel env add SESSION_SECRET production
   vercel env add GOOGLE_CLIENT_ID production
   vercel env add GOOGLE_CLIENT_SECRET production
   vercel env add NEXT_PUBLIC_BASE_URL production
   vercel env add ANTHROPIC_API_KEY production
   ```
   NOTE: If the CLI prompts for values interactively, this will be a checkpoint. Alternatively, the user can add these in the Vercel dashboard. Present both options.

7. Deploy to production: `vercel --prod`

8. Record the production URL from the deploy output.

IMPORTANT: If `vercel` CLI requires interactive authentication, pause and instruct the user to run `vercel login` first. If environment variable setup requires interactive input, present the dashboard alternative and let the user set them there, then re-deploy.
  </action>
