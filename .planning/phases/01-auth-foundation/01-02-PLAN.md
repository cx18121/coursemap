       - Store encrypted tokens: INSERT into oauthTokens with role="personal", ON CONFLICT (userId, role) UPDATE access token + expiry + email. CRITICAL per Pitfall 2: preserve existing encryptedRefreshToken if new tokens don't include a refresh token (hasRefreshToken() returns false)

       - Delete the OAuth state/verifier cookies
       - Redirect to `/`

       - Store state + codeVerifier in cookies with prefix "link_" to avoid collision with login cookies
       - Redirect to URL

    5. Create `src/app/link/school-google/callback/route.ts` (GET handler):
       - Read link_oauth_state + link_code_verifier from cookies

       - Redirect to `/`
