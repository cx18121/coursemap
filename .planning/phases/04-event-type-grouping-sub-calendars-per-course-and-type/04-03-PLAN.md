<objective>
Extend syncCanvasEvents to accept a typeGroupingEnabled flag and route events to per-(course, type) sub-calendars when enabled.

Purpose: The routing change is the core behavioral change of this phase. When the flag is false (default), sync behaves identically to Phase 2. When true, events fan out into type sub-calendars. The bulk-dedup pattern is preserved: one events.list call per sub-calendar (now one per (course, type) bucket instead of one per course when enabled).
Output: gcalSync.ts handles both routing modes. New test cases in gcalSync.test.ts verify the type-routing branch.
</objective>
