## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge padding (`px-2 py-0.5` on event count chip) |
| sm | 8px | Intra-card spacing, gap between tab strip buttons |
| md | 16px | Card internal padding (`p-4`), section inner padding |
| lg | 24px | Gap between major layout blocks (stat card row → countdown hero) |
| xl | 32px | Tab content top padding |
| 2xl | 48px | Between Overview sections (hero → stat card row → sync area) |
| 3xl | 64px | Page-level top/bottom padding |

Exceptions:
- Drawer panel width: `max-w-md` (448px) — fixed by drawer design, not grid-multiple
- Course card grid gap: `gap-3` (12px) — existing project convention for compact card grids; matches `RESEARCH.md` Pattern 4
- Tab strip `border-b` indicator offset: `-mb-px` — 1px overlap for underline visual; not a spacing token

Source: Inferred from existing components (`p-4`, `gap-3`, `px-2 py-0.5`) and RESEARCH.md architecture patterns. Standard 8-pt scale otherwise.

## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Display | 24px (`text-2xl`) | 600 (`font-semibold`) | 1.2 | Dashboard `<h1>` heading |
| Heading | 12px (`text-xs`) | 600 (`font-semibold`) | 1.2 | Section labels: uppercase tracking-wider (e.g. "CANVAS COURSES", bucket headers) |
| Body | 14px (`text-sm`) | 500 (`font-medium`) for interactive; 400 for descriptive | 1.5 | Course names in cards, event list items, panel text, drawer header |
| Label | 12px (`text-xs`) | 400 (`font-normal`) | 1.4 | Event counts ("35 events"), timestamps, secondary metadata |

Notes:
- Only two weights are used in practice: 400 (normal) and 600 (semibold). `font-medium` (500) is used for interactive element labels as a midpoint but maps to the 400/600 binary.
- Tailwind's `text-xs` = 12px, `text-sm` = 14px, `text-2xl` = 24px — these are the only four sizes in use.

Source: Direct read of `SyncDashboard.tsx`, `CountdownPanel.tsx`, `CourseAccordion.tsx`. No new sizes introduced in Phase 8.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#0c0a09` (`--background`) | Page background, body surface |
| Secondary (30%) | `#18181b` (`--color-surface`) / `#1e1e22` (`--color-surface-raised`) | Cards, drawer panel, tab content background (`bg-white/10 backdrop-blur-lg`) |
| Accent (10%) | `#f59e0b` (`--color-accent`) | Focus rings only; stat card active state uses `indigo-400` ring (see below) |
| Destructive | `red-400` (Tailwind) | Inline error messages (e.g. load error `text-red-400`) |

Accent reserved for:
- `*:focus-visible` outline ring (established global rule in `globals.css`)
- Do NOT use `--color-accent` (amber) for active tab indicator, stat card ring, or drawer close button

Active/selected state uses `indigo-400` (not the amber accent token). This is the established project convention from RESEARCH.md Pattern 1 and Pattern 2:
- Active tab: `border-b-2 border-indigo-400`
- Active stat card: `ring-1 ring-indigo-400/60`
- Hover surfaces: `bg-white/[0.15]` (up from `bg-white/10`)
- Disabled / inactive text: `--color-text-secondary` (`rgba(250,250,249,0.55)`)

Border token: `--color-border` = `rgba(255,255,255,0.06)` for card edges; `--color-border-hover` = `rgba(255,255,255,0.12)` on hover.

Source: `globals.css` `@theme` block verified directly.

## Component Inventory

New components introduced in this phase. Each must match the glassmorphic pattern: `bg-white/10 backdrop-blur-lg rounded-2xl border border-[--color-border]`.

### Tab Strip

- Location: Near top of dashboard content area, below `<h1>` heading
- Tabs: "Overview" | "Courses"
- Indicator style: underline — `border-b-2 border-indigo-400 -mb-px` on active tab
- Inactive tab: `text-[--color-text-secondary]`, hover `text-[--color-text-primary]`
- Container: `flex gap-1 border-b border-[--color-border]`
- Behavior: `activeTab` state, always resets to `'overview'` on page load — no localStorage

### StatCard

Dimensions: roughly equal thirds — `grid grid-cols-3 gap-3`

| State | Styling |
|-------|---------|
| Default | `bg-white/10 backdrop-blur-lg rounded-2xl border border-[--color-border] p-4 cursor-pointer` |
| Hover | `bg-white/[0.15]` |
| Active (expanded) | adds `ring-1 ring-indigo-400/60` |

Contents per card:
- Label (top): 12px, weight 400, `text-[--color-text-secondary]` — "Deadlines" / "Synced" / "Conflicts"
- Value (center): 24px, weight 600, `text-[--color-text-primary]` — the count number
- Chevron/indicator (optional, bottom-right): 12px inline SVG arrow

Expanded detail: full-width section rendered below the 3-column card row. Only one expanded at a time. Renders the existing panel component unchanged (`CountdownPanel`, `DedupePanel`, `ConflictPanel`).

### CourseCard

Grid: `grid grid-cols-2 sm:grid-cols-3 gap-3`

Card layout (top-to-bottom):
1. Top row: 16px color swatch (`w-4 h-4 rounded-full`) LEFT + checkbox toggle RIGHT (`w-4 h-4 rounded accent-indigo-500`)
2. Course name: 14px, weight 500, `text-[--color-text-primary] line-clamp-2`
3. Event count: 12px, weight 400, `text-[--color-text-secondary]` — format: "{N} events"

Toggle checkbox uses `e.stopPropagation()` to prevent card click opening drawer.

### CourseDrawer

- Rendered via `createPortal(content, document.body)` — required to escape `backdrop-blur-lg` stacking context
- Width: `w-full max-w-md` (448px)
- Position: `fixed top-0 right-0 h-full`
- z-index: drawer panel `z-50`, backdrop `z-40`
- Background: `bg-[--color-surface] border-l border-[--color-border] shadow-2xl`
- Animation: CSS `transition-transform duration-200` on `translate-x-full` → `translate-x-0` (do NOT use `tailwindcss-animate` — not installed in this project)
- Backdrop: `fixed inset-0 bg-black/40 z-40` — click closes drawer
- Dismiss: Escape key (`keydown` listener, same pattern as `ColorPicker`)
- Header: course name (14px, weight 600) + close button (inline SVG X, 16px)
- Body: `p-4` padding, `overflow-y-auto`, contains existing `CourseAccordion` content

### Countdown Hero (Overview tab)

The countdown deadlines section occupies the top of the Overview tab with largest visual weight — it is not inside a collapsible stat card. It renders `CountdownPanel` directly, not through the stat card expand mechanism.

Layout: full-width card, `bg-white/10 backdrop-blur-lg rounded-2xl border border-[--color-border] p-4`, no collapse affordance.

### Inline SyncButton Area

- No fixed positioning — renders in normal document flow below the stat card row
- SyncButton receives `inline` prop (boolean) to switch from `fixed bottom-0` wrapper to a flow `div`
- SyncSummary renders inline below SyncButton, also with `inline` prop
- Remove `pb-32` bottom padding from SyncDashboard outer container (was compensating for fixed bar)

## Interaction Contract

| Interaction | Behavior |
|-------------|----------|
| Click tab | `setActiveTab(tab)` + `setOpenCourseDrawer(null)` (close any open drawer) |
| Click stat card | Toggle `expandedPanel` — same panel click collapses; different panel replaces |
| Click CourseCard body | `setOpenCourseDrawer(courseName)` |
| Click CourseCard checkbox | Toggle enabled state via `onToggle`; `e.stopPropagation()` — does NOT open drawer |
| Click drawer backdrop | `setOpenCourseDrawer(null)` |
| Press Escape | `setOpenCourseDrawer(null)` when drawer is open |
| Sync completes | `key={syncVersion}` forces ConflictPanel remount; stat card counts re-fetch |

Loading states:
- Course card grid: skeleton cards (same `bg-white/10 rounded-2xl` shape, no content, opacity pulse) while `loadingCourses === true`
- Stat card values: show "—" until mount fetch resolves; show count number after
