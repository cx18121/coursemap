  <name>Task 2: Create syncFilter, colorAssignment, and titleCleanup services with tests</name>
  <files>src/services/syncFilter.ts, src/services/syncFilter.test.ts, src/services/colorAssignment.ts, src/services/colorAssignment.test.ts, src/services/titleCleanup.ts, src/services/titleCleanup.test.ts</files>
  <behavior>
    syncFilter:
    - filterEventsForSync(userId, groupedEvents) returns only events from enabled courses that are not overridden as disabled
    - Course with enabled=false excludes ALL its events
    - Event with an eventOverride row where enabled=false is excluded even if course is enabled
    - Course with no row in courseSelections defaults to enabled (new course auto-include per CONTEXT.md)
    - Event with no row in eventOverrides defaults to enabled

    colorAssignment:
    - assignCourseColors(userId, courseNames[]) assigns distinct colorIds from Google's 11 event colors (1-11)
    - Courses already in DB keep their existing colorId
    - New courses get the next unused colorId (round-robin if >11 courses, but user said max 11)
    - Returns Record<courseName, colorId>

    titleCleanup:
    - getCleanedTitle(rawTitle) checks eventTitleCache first, returns cached if found
    - If not cached and ANTHROPIC_API_KEY is set, calls claude-3-haiku for cleanup, caches result
    - If not cached and ANTHROPIC_API_KEY is NOT set, falls back to regex: strip trailing [...] brackets, strip leading "Submit " prefix
    - cleanTitlesBatch(titles[]) processes multiple titles efficiently (checks cache in bulk, calls AI only for uncached)
  </behavior>
  <action>
Create three service modules in `src/services/`:

**syncFilter.ts:**
- Export `filterEventsForSync(userId: number, groupedEvents: GroupedEvents): Promise<CanvasEvent[]>`
- Query `courseSelections` for this userId to get enabled/disabled courses
- Query `eventOverrides` for this userId to get per-event exclusions
- For courses with no DB row, treat as enabled (auto-include new courses)
- For events with no override row, treat as enabled
- Return flat array of CanvasEvent[] that pass both filters
- Also export `ensureCourseSelections(userId: number, courseNames: string[]): Promise<void>` that upserts courseSelections rows for any new courses (so they appear in the UI)

**colorAssignment.ts:**
- Export `GOOGLE_CALENDAR_COLORS: Record<string, string>` mapping colorId to name (1=Lavender through 11=Tomato)
- Export `assignCourseColors(userId: number, courseNames: string[]): Promise<Record<string, string>>`
- Query existing courseSelections for userId, collect used colorIds
- For new courses without a colorId, assign the next unused ID from 1-11 (round-robin)
- Upsert courseSelections with assigned colorId
- Return map of courseName -> colorId

**titleCleanup.ts:**
- Export `getCleanedTitle(rawTitle: string): Promise<string>`
- Check `eventTitleCache` for exact match on originalTitle
- If cached, return cleanedTitle
- If ANTHROPIC_API_KEY env var is set, call `@anthropic-ai/sdk` with claude-3-haiku-20240307: system prompt strips course tags, boilerplate prefixes, truncates to 60 chars
- If ANTHROPIC_API_KEY is NOT set, use regex fallback: remove trailing `[...]` bracket content, remove leading "Submit " or "Submit Assignment: "
- Cache result in eventTitleCache before returning
- Export `cleanTitlesBatch(rawTitles: string[]): Promise<Record<string, string>>` that bulk-checks cache and only calls AI for uncached titles

Write tests for all three services. Mock the DB layer using jest.mock for drizzle queries. Mock @anthropic-ai/sdk for titleCleanup tests. Use the existing jest.config.ts and ts-jest setup. Follow the established test pattern from icalParser.test.ts.
